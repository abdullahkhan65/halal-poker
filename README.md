## Setup

### 1. Create a Supabase project

Go to [supabase.com](https://supabase.com), create a project, then grab:
- **Database URL** ‚Äî Settings ‚Üí Database ‚Üí Connection string (URI mode)
- **Supabase URL** ‚Äî Settings ‚Üí API ‚Üí Project URL
- **Service Role Key** ‚Äî Settings ‚Üí API ‚Üí service_role key (keep secret)

### 2. Configure environment

Edit `backend/.env`:

```env
DATABASE_URL="postgresql://postgres:[password]@db.[ref].supabase.co:5432/postgres"
SUPABASE_URL="https://[ref].supabase.co"
SUPABASE_SERVICE_KEY="eyJ..."
JWT_SECRET="generate-a-long-random-string"
PORT=3001
```

### 3. Run database migrations

```bash
cd backend
npx prisma migrate dev --name init
```

### 4. Make yourself an admin

In Supabase Table Editor ‚Üí `User` table ‚Üí find your row ‚Üí set `isAdmin = true`.

Or via SQL:
```sql
UPDATE "User" SET "isAdmin" = true WHERE email = 'you@example.com';
```

### 5. Install dependencies

```bash
npm install               # root (installs concurrently)
npm install --prefix backend
npm install --prefix frontend
```

### 6. Start

```bash
npm run dev
```

- Frontend: [http://localhost:5173](http://localhost:5173)
- Backend: [http://localhost:3001](http://localhost:3001)

## How It Works

### Auth Flow

1. User enters email ‚Üí Supabase sends magic link
2. User clicks link, copies the token from the URL (`#access_token=...`)
3. Pastes token into the app ‚Üí app verifies with Supabase ‚Üí issues 30-day JWT
4. **New users** must also provide an invite code (generated by admin)

### Invite System

Only admin can generate invite codes. Each code is single-use.

Admin Panel ‚Üí "Generate" ‚Üí share the 10-char code with your friend.

### Starting a Game (Online)

1. Any logged-in player creates a table in the Lobby
2. Others join the table
3. Any player clicks **Start Game** (needs ‚â• 2 players)
4. Everyone starts with **20,000 chips**
5. Texas Hold'em proceeds: Pre-Flop ‚Üí Flop ‚Üí Turn ‚Üí River ‚Üí Showdown
6. Winner gets the pot, hand rank shown on screen

### Recording an Offline Session (Admin)

1. Admin Panel ‚Üí **Create Session** ‚Üí set label, date, toggle Offline
2. Enter ending chip counts for each player
3. Click **Finalize** ‚Äî profits auto-calculated (`endChips - 20000`) and saved
4. Leaderboard updates immediately

### Leaderboard

Sorted by `totalEarnings` (cumulative profit across all sessions). Each session shows a breakdown with profit/loss per player.

## API Reference

|--------|------|------|------|

### Users

### Sessions
| Method | Path | Auth |
|--------|------|------|
| GET | `/api/sessions` | JWT |
| GET | `/api/sessions/:id` | JWT |
| POST | `/api/sessions` | JWT + Admin |
| POST | `/api/sessions/:id/finalize` | JWT + Admin |

### Socket.IO Events

| Emit | Payload | Response |
|------|---------|----------|
| `join_table` | `{ tableId, chips? }` | `GameState` |
| `leave_table` | `{ tableId }` | ‚Äî |
| `create_table` | `{ name }` | `{ id, name }` |
| `get_tables` | ‚Äî | `TableInfo[]` |
| `start_game` | `{ tableId }` | `GameState` |
| `player_action` | `{ tableId, action, raiseAmount? }` | `GameState` |

Actions: `fold` `check` `call` `raise`

Server broadcasts `game_state` to all players in a table room after every action.

### Voice Chat (WebRTC ‚Äî client emits, server relays)

| Emit | Payload | Server broadcasts |
|------|---------|-------------------|
| `voice_join` | `{ tableId }` | `voice_peer_joined` ‚Üí all others in room |
| `voice_offer` | `{ tableId, targetId, offer }` | `voice_offer` ‚Üí all others |
| `voice_answer` | `{ tableId, targetId, answer }` | `voice_answer` ‚Üí all others |
| `voice_ice` | `{ tableId, targetId, candidate }` | `voice_ice` ‚Üí all others |

Server also emits `voice_peer_left` when a player disconnects or leaves the table.
